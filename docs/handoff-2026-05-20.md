## Key Architecture Notes (for context)

- **Orders table**: Has `customer_phone` directly, no `customer_id` FK — don't join customers table
- **Rider assignment**: FCFS (first-come-first-served) via atomic DB update, NOT manual dispatch
- **Manual dispatch**: REMOVED from codebase — `dispatchOrder()` rejects when no rider assigned
- **Tailwind v4**: Upgraded to 4.3.0; `@source not "../node_modules"` prevents oxide scanner from reading HTML-encoded classes in node_modules
- **Google Maps**: `distance.ts` uses server-side `GOOGLE_MAPS_API_KEY`; `OrderTracker.tsx` uses client-side `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` for iframe embed
- **Rider location**: `startRiding()` now writes GPS to `riders.current_location` when lat/lng params provided; dashboard tracks `lastLat`/`lastLng` refs from `watchPosition`
